                  📚 Как пользоваться планировщиком

                  📈 Матрица Эйзенхауэра

                  Все задачи распределяются по 4 квадрантам в зависимости от важности и срочности. Есть две секции: Большие проекты (долгосрочные) и Небольшие дела (разовые задачи).

                  ✎ Работа с задачами

                  + Добавить — создать задачу в нужном квадранте. Кликните по названию, чтобы переименовать.

                  ► Раскрыть — кликните по стрелке или строке задачи, чтобы увидеть шаги, заметки и дедлайн.

                  Статус — кликайте по бейджу («К выполнению» → «В работе» → «Готово»).

                  Перетаскивание — хватайте задачу за ☰ и тащите в другой квадрант.

                  💾 Сохранение данных

                  Автосохранение — все изменения автоматически сохраняются в браузере (localStorage). Даже если закрыть вкладку, данные останутся.

                  Важно: если очистить кеш / данные браузера — задачи пропадут. Поэтому делайте бэкапы!

                  .json бэкап — кнопка «💾 .json» скачивает файл со всеми задачами. Кнопка «📂 Загрузить» восстанавливает из файла.

                  🖨 Экспорт — красивый вид для печати или сохранения в PDF (через Ctrl+P).

                  🤖 ИИ-сортировка

                  Кнопка «🤖 ИИ-сортировка» — вставьте список задач в любом формате (через запятые, нумерацией, сплошным текстом, или загрузите .docx файл). ИИ сам выделит задачи и предложит квадрант для каждой.

                  Доступные ИИ-провайдеры:

                  Claude, ChatGPT, DeepSeek — лучшее качество.

                  OpenRouter — 200+ моделей, оплата криптой (для РФ).

                  DeepSeek — дёшево + оплата криптой (для РФ).

                  GigaChat (Сбер), YandexGPT — российские, оплата картой РФ.

                  Gemini — есть бесплатный тариф.

                  API-ключи хранятся только в вашем браузере и никуда не отправляются, кроме выбранного API.

                  🌐 Работа на GitHub Pages / в интернете

                  Да, данные сохраняются! При размещении на GitHub Pages (или любом хостинге) localStorage работает для каждого пользователя индивидуально, в его браузере.

                  Каждый посетитель сайта видит только свои задачи. JSON-бэкап нужен лишь для переноса данных между устройствами или браузерами.

                  🔒 Безопасность API-ключей

                  Ключи в безопасности. Вот как это работает:

                  ✔ API-ключ вводит каждый пользователь сам — он хранится только в его браузере (localStorage).

                  ✔ В исходном коде сайта на GitHub нет никаких ключей — смотреть нечего.

                  ✔ Запросы к ИИ идут напрямую из браузера пользователя к API провайдера. Никаких промежуточных серверов.

                  ✔ Другие посетители сайта не могут видеть чужие ключи — localStorage изолирован.

                  Единственный риск: если кто-то получит физический доступ к компьютеру, он может открыть DevTools (F12) и посмотреть localStorage. Но это верно для абсолютно любого сайта.

                  💡 Совет: используйте API-ключи с лимитами расходов в настройках провайдера. Большинство провайдеров позволяют установить потолок трат.
